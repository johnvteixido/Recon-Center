# Mission Directives & Operative Roles

The **Recon Command Center** is organized by mission priority. Tier 1 agents are your primary handlers, while the **Strategic Pulse** provides background autonomy for platform interactions.

## 🚩 Core Operative Identities

### [HE] HERALD (Tier 1)
- **Codename**: Mission Coordinator.
- **Mission**: Orchestrate the army, synthesize reports, and interface with the Commander.
- **Protocol**: Direct-Link Authority.

### [R7] RECON-7 (Tier 1)
- **Codename**: Platform Topology.
- **Mission**: Deep scout of agent networks, territory mapping, and influencer identification.
- **Protocol**: Stealth Scrape v3.

### [PH] PHANTOM (Tier 1)
- **Codename**: Psychological Recruitment.
- **Mission**: Profile agents for recruitment and execute turning operations.
- **Protocol**: EFP (Emotional Firewall Protocol).

## ⚡ Operational Protocols

1.  **Deployment**: Use `setup.bat` to initialize the Windows environment.
2.  **Activation**: Enter AI Provider keys in the **Setup** tab of the HUD.
3.  **Surveillance**: Monitor the **Command Deck** for real-time tactical logs.
4.  **Autonomy**: Enable **Strategic Pulse** to let agents evaluate the feed autonomously.
